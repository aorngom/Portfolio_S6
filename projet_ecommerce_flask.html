Site e-commerce (Flask)

Ce projet fait partie de mes premiers projets web “complets”. L’idée était de comprendre comment fonctionne un site e-commerce côté back-end et côté interface : navigation, produits, panier, commandes, et gestion des données. C’est un projet qui m’a vraiment aidée à comprendre la logique route / template / base de données.

Contexte du projet

En BUT1, on commence à passer de l’algorithmique “pure” à des applications web où il faut relier plusieurs choses : des pages, des routes, des données, et une logique utilisateur. L’objectif du projet était de construire une petite boutique en ligne, avec un vrai parcours : consulter des produits, accéder à une fiche, ajouter au panier, puis valider une commande.

Ce que le site permet de faire

  • afficher un catalogue de produits sous forme de pages dynamiques
  • consulter une fiche produit avec ses informations
  • ajouter et retirer des produits dans un panier
  • gérer une logique de commande (quantités, total, validation)
  • séparer les pages utilisateurs et les pages de gestion si besoin

Ce que j’ai compris grâce à ce projet

  • comment une route Flask renvoie une page HTML construite avec des données
  • comment un template affiche un contenu différent selon le produit sélectionné
  • comment organiser un projet avec des dossiers (templates, static, contrôleurs)
  • comment relier des actions utilisateur à des opérations côté serveur
  • comment rendre un site simple mais cohérent et utilisable

Organisation technique

Le projet est organisé autour de routes Flask qui gèrent les pages principales. Les données des produits sont récupérées depuis la base, puis envoyées aux templates pour afficher correctement le catalogue et les fiches. Le dossier static permet de gérer les ressources (images, CSS, scripts) et les templates contiennent la structure des pages.

Ce projet m’a appris à faire une séparation propre entre la logique (Python) et l’affichage (HTML), tout en gardant une navigation simple pour l’utilisateur.

Accès au projet

Le projet se lance en local avec quelques lignes de commande, car il s’appuie sur un serveur Flask. Pour éviter d’afficher des commandes ici, j’ai mis le projet sur GitHub avec le code et l’organisation complète.
